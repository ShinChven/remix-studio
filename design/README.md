# Design Docs

This directory contains implementation design notes for AI agents and developers.

Read the relevant file here before changing a subsystem with non-obvious architecture, data flow, or invariants.

These files are internal design references, not end-user documentation.

Suggested entry points:

- `system-overview.md` for a high-level map of the main subsystems
- `image-handling.md` for image upload, storage, and signing behavior
- `storage-analytics.md` for quota and storage analysis behavior
- `queue-architecture-design.md` and `runninghub-polling-strategy.md` for generation queue behavior
